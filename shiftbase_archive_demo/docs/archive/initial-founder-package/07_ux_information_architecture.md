# UX와 정보 구조

## UX 원칙

Shiftbase의 UX는 사용자가 보는 사람이 아니라 조작하는 사람이 되도록 설계해야 합니다.

핵심 원칙:

- 한 화면에서 다음 행동이 명확해야 함
- 실습이 긴 설명보다 우선해야 함
- 결과물이 계속 쌓여야 함
- 초보자가 길을 잃지 않아야 함
- 실습 중 막히면 힌트를 제공해야 함
- AI 결과는 항상 사용자가 검토하도록 설계해야 함

## 모바일 전략

초기에는 데스크톱 실습을 우선합니다.

모바일에서는 다음만 우선 제공합니다.

- 모듈 탐색
- 설명 읽기
- 자료 다운로드
- 진도 확인
- 간단한 실습 미리보기

복잡한 시뮬레이션은 데스크톱 권장으로 안내합니다.

## 접근성 원칙

- 버튼 텍스트는 명확하게 작성
- 색상만으로 상태를 구분하지 않음
- 진행 단계에는 텍스트 상태를 함께 표시
- 키보드 이동을 고려
- 입력 오류에는 구체적인 설명 제공

## UX 핵심 문장

Shiftbase의 모든 화면은 사용자가 다음 질문에 답하게 해야 합니다.

지금 내가 무엇을 해야 하나
왜 이 행동이 업무전환에 필요한가
이 실습이 끝나면 무엇을 가져갈 수 있나

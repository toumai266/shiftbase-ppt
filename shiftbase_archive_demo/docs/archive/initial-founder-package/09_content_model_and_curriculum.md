# 커리큘럼과 콘텐츠 모델

## 콘텐츠 원칙

Shiftbase의 콘텐츠는 지식 전달보다 업무 전환을 목표로 합니다.

각 콘텐츠는 다음을 가져야 합니다.

- 명확한 업무 상황
- 학습자가 수행할 행동
- AI가 들어갈 지점
- 검토 기준
- 실습 결과물
- 현업 적용 템플릿

## 콘텐츠 계층

### Category

업무 영역 또는 산업 영역입니다.

예시:

- 소규모 업체를 위한 AX
- 사무업무 자동화
- 창업 문서 자동화
- AI 보안과 운영
- 바이브코딩
- 제조업을 위한 AX

### Course 또는 Module

학습자가 완료할 수 있는 단위입니다.

예시:

- 고객문의 자동응답과 상담 분류
- 예약 문의를 일정으로 바꾸기
- 리뷰 분석과 답변 자동화

### Lesson

개념 설명 또는 짧은 읽기 단위입니다. 영상이 없더라도 카드형 설명, 예시, 미니 퀴즈로 구성할 수 있습니다.

### Simulation

웹 인터페이스 기반 실습 단위입니다.

### Template

현업에 가져갈 수 있는 결과물입니다.

예시:

- 문의 유형 기준표
- FAQ 템플릿
- 응답 체크리스트
- 상담 기록 양식
- 주간 운영 보고서

### Checklist

사용자가 결과물을 검토하는 기준입니다.

## 초기 카테고리 설계

### 소규모 업체를 위한 AX

대상:

- 카페
- 미용실
- 학원
- 클리닉
- 온라인 판매자
- 소규모 사무실

모듈:

1. 고객문의 자동응답과 상담 분류
2. 예약 문의를 일정으로 바꾸기
3. 리뷰 분석과 답변 자동화
4. 자동화된 상담 기록 관리
5. 사장님을 위한 주간 운영 보고서 만들기
6. 견적서와 안내문 빠르게 작성하기

### 사무업무 자동화

모듈:

1. 회의 메모를 보고서로 바꾸기
2. 이메일 답변 기준 만들기
3. 주간업무 보고서 자동화
4. 자료 요약과 액션 아이템 정리
5. 엑셀과 문서 업무 정리

### 창업 문서 자동화

모듈:

1. 아이디어 메모를 사업계획서 초안으로 바꾸기
2. 고객 문제 정의하기
3. 경쟁 서비스 분석하기
4. 제안서와 소개서 초안 만들기
5. 발표자료 흐름 만들기

### AI 보안과 운영

모듈:

1. 회사에서 AI에 넣으면 안 되는 정보
2. AI 사용 정책 만들기
3. 프롬프트 인젝션 쉽게 이해하기
4. 직원용 AI 사용 체크리스트
5. AI 결과 검토와 로그 관리

### 바이브코딩

모듈:

1. 아이디어를 PRD로 바꾸기
2. 화면 요구사항 작성하기
3. AI와 함께 MVP 만들기
4. 생성된 코드 검토하기
5. 배포 전 체크리스트

## 모듈 표준 구성

모든 모듈은 다음 구조를 기본으로 합니다.

1. 문제 상황
2. 업무흐름 설명
3. AI 적용 포인트
4. 실습 시나리오
5. 사용자 액션
6. 검증과 피드백
7. 결과물 저장
8. 템플릿 다운로드
9. 현업 적용 체크리스트

## 콘텐츠 제작 체크리스트

콘텐츠를 만들 때 다음을 확인합니다.

- 대상 사용자가 명확한가
- 실제 반복 업무인가
- 입력값과 출력값이 정해져 있는가
- AI가 들어갈 지점이 명확한가
- 사람이 검토해야 할 기준이 있는가
- 결과물이 업무에 바로 쓰일 수 있는가
- 실습 시간이 15분에서 60분 안에 들어오는가
- 예시 데이터가 충분한가

## 난이도 체계

### 입문

- AI 개념을 몰라도 가능
- 선택과 분류 중심
- 결과물은 템플릿 수준

### 초급

- 기본 프롬프트와 체크리스트 사용
- 간단한 문서나 답변 작성
- 실무 적용 계획 포함

### 중급

- 업무흐름 설계 포함
- 여러 단계 연결
- AI 결과 평가와 수정 포함

### 고급

- 시스템 설계
- API 또는 자동화 도구 연결
- 보안과 운영 기준 포함

## 콘텐츠 품질 기준

좋은 모듈은 다음 조건을 만족합니다.

- 사용자가 자신의 업무를 떠올릴 수 있음
- 실습 화면이 실제 업무와 닮아 있음
- 설명보다 행동이 많음
- 실습 결과물이 명확함
- AI 출력이 과장되지 않음
- 현업 적용 시 주의사항을 포함함

## 콘텐츠 생산 파이프라인

4. 예시 데이터 작성
5. 실습 단계 설계
6. 템플릿 작성
7. 검증 기준 정의
8. 인터페이스 구현
9. 파일럿 테스트
10. 수정 후 출시

# AI 서비스 설계

## AI 서비스의 역할

Shiftbase에서 AI는 학습 경험의 보조 엔진입니다. AI가 제품의 전부가 되어서는 안 됩니다. 실습 인터페이스와 검증 로직이 기본이고, AI는 분류, 초안, 피드백, 힌트를 보조합니다.

## 기본 원칙

- AI 없이도 실습은 진행 가능해야 함
- AI 결과는 사용자가 검토해야 함
- 실제 민감정보 입력은 최소화
- 샘플 데이터와 가상 데이터를 기본 사용
- AI 출력은 정답이 아니라 초안으로 표시
- 로그는 품질 개선과 비용 관리 목적으로 저장

## AI 기능 범위

### 1차 기능

- 고객문의 유형 분류
- 답변 초안 생성
- 답변 품질 평가
- 누락 정보 감지
- 실습 힌트 제공

### 2차 기능

- 사용자 결과 리포트 요약
- 업무별 템플릿 추천
- 현업 적용 계획 피드백
- 기업교육 수강자 리포트 요약

## 기능별 설계

### 문의 유형 분류

입력:

- 문의 내용
- 채널
- 업종
- 기존 유형 목록

출력:

- 추천 유형
- 보조 유형
- 우선순위
- 필요한 추가 정보
- 신뢰도

예시 출력 구조:

```yaml
primary_category: 예약
secondary_category: 가격 문의
priority: 보통
required_info:
  - 날짜
  - 인원
  - 서비스 종류
confidence: 92
reason: 날짜와 인원, 가격 질문이 함께 포함되어 있음
```

### 답변 초안 생성

입력:

- 문의 내용
- 분류 결과
- 업체 유형
- 응답 톤
- 필수 안내 정보
- 금지 표현

출력:

- 답변 초안
- 포함된 핵심 정보
- 누락 가능 정보
- 검토 필요 항목

### 답변 품질 평가

입력:

- 문의 내용
- 답변 초안
- 체크리스트

출력:

- 친절한 톤 여부
- 핵심 정보 포함 여부
- 다음 행동 제시 여부
- 민감정보 제외 여부
- 과장 표현 여부
- 총점

### 힌트 제공

입력:

- 현재 단계
- 사용자 액션
- 오답 또는 누락 상태

출력:

- 짧은 힌트
- 관련 개념
- 다음 행동 제안

## API 구조 초안

### classify inquiry

경로:

```text
POST /ai/classify-inquiry
```

입력:

```yaml
inquiry_text: 안녕하세요. 이번 주 토요일 2시에 2명 예약 가능한가요? 가격도 궁금합니다.
channel: 카카오톡
business_type: 미용실
categories:
  - 예약
  - 가격 문의
  - 운영 안내
  - 불만
  - 기타
```

출력:

```yaml
primary_category: 예약
secondary_category: 가격 문의
priority: 보통
required_info:
  - 날짜
  - 인원
  - 서비스 종류
confidence: 92
```

### generate response draft

경로:

```text
POST /ai/generate-response-draft
```

입력:

```yaml
inquiry_text: 안녕하세요. 이번 주 토요일 2시에 2명 예약 가능한가요? 가격도 궁금합니다.
category: 예약, 가격 문의
business_type: 미용실
tone: 친절하고 간결함
constraints:
  - 확정되지 않은 가격을 단정하지 않음
  - 개인정보를 요구하지 않음
```

출력:

```yaml
draft: 안녕하세요. 문의 주셔서 감사합니다. 이번 주 토요일 2시 2명 예약 가능 여부를 확인해드릴게요. 원하시는 시술 종류에 따라 가격이 달라질 수 있어 메뉴와 예상 금액을 함께 안내드리겠습니다.
checkpoints:
  - 예약 가능 여부 확인 필요
  - 서비스 종류 확인 필요
  - 가격 단정 표현 없음
```

### evaluate response

경로:

```text
POST /ai/evaluate-response
```

출력:

```yaml
score: 88
checks:
  friendly_tone: pass
  core_info: partial
  next_action: pass
  sensitive_info: pass
  overclaim: pass
feedback: 가격 안내 기준을 조금 더 명확히 작성하면 좋습니다
```

## 프롬프트 관리

프롬프트는 코드에 하드코딩하지 말고 버전 관리합니다.

권장 필드:

- prompt_key
- version
- purpose
- system_instruction
- user_template
- output_schema
- status
- created_at

초기에는 파일 기반으로 시작해도 됩니다.

## AI 로그

저장할 항목:

- 사용자 id
- 실습 id
- 기능명
- 입력 요약
- 출력 요약
- 모델명
- 토큰 수
- 비용 추정
- 에러 여부

민감정보가 포함될 수 있으므로 입력 전문 저장은 기본 비활성화합니다.

## 비용 통제

- 무료 사용자는 AI 호출 횟수 제한
- 동일 입력 반복 호출 캐시
- 긴 입력 요약 후 처리
- AI 실패 시 예시 결과 대체
- 관리자에서 월별 호출량 확인

## 품질 통제

- 샘플 문의 세트로 정기 평가
- 카테고리 분류 정확도 측정
- 답변 품질 체크리스트 통과율 측정
- 사용자 수정률 측정
- 잘못된 답변 신고 기능 도입

## 안전장치

- 의료, 법률, 금융 상담으로 오해될 수 있는 응답 제한
- 개인정보 요구 제한
- 가격, 예약 확정 등은 실제 시스템 확인 필요 안내
- AI 답변은 발송 전 사람이 검토하도록 설계

## 핵심 판단

초기 AI 기능은 화려할 필요가 없습니다. 사용자가 업무흐름을 이해하고 결과물을 수정할 수 있게 돕는 보조 기능이면 충분합니다.

# 운영과 관리자 기능

## 운영 원칙

초기에는 완전한 관리자 시스템보다 빠른 콘텐츠 수정과 사용자 피드백 수집이 중요합니다.

초기 운영 방식:

- 콘텐츠는 MDX와 YAML로 관리
- 데이터는 Supabase 콘솔에서 확인
- 파일럿 사용자는 수동으로 관리
- 유료화 전까지 결제는 단순하게 운영

## 관리자 기능 단계

### 1단계: 수동 운영

도구:

- Git 기반 콘텐츠 관리
- Supabase Dashboard
- 폼 기반 신청 관리
- 수동 이메일 안내

관리 항목:

- 사용자 목록
- 실습 진행률
- 다운로드 이력
- 파일럿 신청자
- 피드백

### 2단계: 내부 Admin

기능:

- 카테고리 생성과 수정
- 모듈 생성과 수정
- 실습 시나리오 확인
- 템플릿 업로드
- 사용자 진도 확인
- 수강권 부여
- 쿠폰 발급

### 3단계: B2B Admin

기능:

- 조직 생성
- 구성원 초대
- 수강 과정 배정
- 코호트 관리
- 진도 리포트
- 수료증 발급
- 조직별 결과 요약

## 콘텐츠 운영 프로세스

1. 업무 문제 선정
2. 시나리오 초안 작성
3. 예시 데이터 작성
4. UX 흐름 작성
5. 템플릿 제작
6. 실습 구현
7. 내부 테스트
8. 파일럿 테스트
9. 수정 후 공개
10. 지표 모니터링

## 사용자 지원

초기 지원 채널:

- 이메일
- 문의 폼
- 실습 중 피드백 버튼

지원 항목:

- 로그인 문제
- 다운로드 문제
- 실습 진행 오류
- AI 결과 오류 신고
- 결제와 환불 문의

## 피드백 수집

실습 완료 후 질문:

- 이 실습이 실제 업무에 도움이 되었나요
- 어떤 부분이 가장 헷갈렸나요
- 어떤 템플릿이 가장 유용했나요
- 추가로 배우고 싶은 업무는 무엇인가요
- 유료라면 적정 가격은 얼마인가요

## 품질 관리

출시 전 체크리스트:

- 모든 버튼 동작 확인
- 실습 단계 완료 조건 확인
- 템플릿 다운로드 확인
- 모바일 최소 화면 확인
- 로그인과 이어하기 확인
- AI 실패 시 대체 결과 확인
- 민감정보 입력 안내 표시

## 실습 데이터 관리

예시 데이터는 다음 기준을 지켜야 합니다.

- 실제 개인정보 사용 금지
- 특정 업체로 오해될 만한 정보 지양
- 업종별로 현실적인 문장 사용
- 불만 문의는 과도하게 자극적이지 않게 작성
- 의료, 법률, 금융 민감 상담은 주의

## 파일 관리

자료 유형:

- PDF 가이드
- XLSX 템플릿
- DOC 또는 MD 템플릿
- ZIP 샘플 자료

초기에는 파일이 적으므로 Supabase Storage로 관리합니다.

## 수료 운영

초기 수료 조건:

- 실습 단계 80퍼센트 이상 완료
- 최종 체크리스트 통과
- 결과 리포트 생성

기업교육 수료 조건:

- 지정 모듈 완료
- 과제 결과 제출
- 관리자 승인 선택

## 장애 운영

자주 발생할 수 있는 문제:

- AI 응답 지연
- 결과 저장 실패
- 다운로드 링크 만료
- 실습 단계 상태 꼬임
- 결제 후 권한 부여 실패

대응:

- 재시도 버튼
- 임시 저장
- 관리자 수동 권한 부여
- AI 예시 결과 대체
- 오류 로그 수집

## 운영 지표

- 일간 활성 사용자
- 실습 시작 수
- 실습 완료 수
- 다운로드 수
- 오류 신고 수
- 평균 지원 응답 시간
- AI 호출 성공률
- 결제 실패율

## 핵심 판단

초기에는 관리자 기능을 많이 만들지 말고, 실습 완성도와 콘텐츠 품질에 시간을 써야 합니다. 관리자 기능은 반복 운영 병목이 생기는 지점부터 만들면 됩니다.

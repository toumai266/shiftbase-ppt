# 제품 범위와 로드맵

## 제품 원칙

Shiftbase는 영상 중심 강의 플랫폼이 아닙니다. 웹사이트 자체가 교육 인터페이스이며, 사용자는 실제 업무 상황을 조작하면서 AI 업무전환을 배웁니다.

초기 제품은 다음 네 가지를 반드시 만족해야 합니다.

- 사용자가 직접 클릭하거나 입력해야 함
- 실습 결과물이 남아야 함
- 실제 업무에 가져갈 템플릿이 있어야 함
- 진도와 결과가 저장되어야 함

## 초기 제품 범위

### 포함

- 브랜드 홈
- 소규모 업체를 위한 AX 카테고리
- 고객문의 자동응답과 상담 분류 모듈
- 고객 문의 워크플로우 실습실
- 로그인
- 진도 저장
- 실습 결과 저장
- 템플릿 다운로드
- 간단한 나의 학습 대시보드

### 제외

- 영상 플레이어
- 강사 마켓플레이스
- 커뮤니티 게시판
- 복잡한 구독권
- 모바일 전용 UI
- 대규모 관리자 페이지
- 모든 카테고리 동시 출시

## MVP 모듈

고객문의 자동응답과 상담 분류

학습자가 수행하는 흐름:

1. 여러 채널에서 들어온 문의 확인
2. 문의 유형 분류
3. 필요한 정보 보강
4. 답변 초안 생성 또는 선택
5. 응답 품질 체크리스트 통과
6. 발송 전 검토
7. 상담 기록 저장
8. FAQ 템플릿과 응답 기준표 다운로드

## 단계별 로드맵

### Phase 0: 기획과 설계

기간 예시: 1주에서 2주

산출물:

- 브랜드와 핵심 카피 확정
- MVP 모듈 확정
- 화면 구조 확정
- DB 스키마 초안
- 실습 시나리오 정의
- 디자인 시스템 기본값 확정

완료 기준:

- 클릭 가능한 핵심 화면 목록이 정리됨
- 실습 단계와 완료 조건이 문서화됨

### Phase 1: 인터페이스 MVP

기간 예시: 3주에서 5주

구현 범위:

- 홈
- 카테고리
- 모듈 상세
- 실습실 화면
- 정적 실습 시나리오
- 템플릿 다운로드
- 진도 상태 임시 저장

완료 기준:

- 영상 없이도 데모 가능
- 사용자 5명 이상이 처음부터 끝까지 실습 완료 가능
- 실습 완료 후 결과 화면 확인 가능

### Phase 2: 저장형 MVP

기간 예시: 2주에서 4주

구현 범위:

- Supabase Auth
- 사용자별 진행률 저장
- 실습 결과 저장
- 나의 학습 대시보드
- 다운로드 이력
- 기본 관리자용 데이터 확인

완료 기준:

- 로그인 후 이어하기 가능
- 실습 결과가 DB에 저장됨
- 템플릿 다운로드 이력이 남음

### Phase 3: AI 보조 기능

기간 예시: 3주에서 5주

구현 범위:

- 문의 유형 분류
- 답변 초안 생성
- 응답 품질 평가
- 누락 정보 감지
- 실습 피드백
- AI 호출 로그와 비용 추적

완료 기준:

- AI가 실패해도 예시 결과로 실습 진행 가능
- 사용자는 AI 결과를 수정하고 저장할 수 있음
- AI 출력 품질을 기본 체크리스트로 검증 가능

### Phase 4: 유료화

기간 예시: 2주에서 4주

구현 범위:

- Toss Payments 연동
- 단건 구매
- 패키지 구매
- 무료 체험과 유료 모듈 분리
- 구매자 권한 관리

완료 기준:

- 결제 후 모듈 접근 가능
- 환불과 접근 권한 처리 가능
- 결제 웹훅으로 주문 상태 업데이트 가능

### Phase 5: B2B 파일럿

기간 예시: 4주에서 8주

구현 범위:

- 조직 계정
- 조직 구성원 관리
- 코호트 생성
- 수강자 진도 리포트
- 수료증 또는 수료 배지
- 기업용 문의 페이지

완료 기준:

- 1개 이상 소규모 기업 또는 단체에 파일럿 제공 가능
- 관리자에게 진도와 완료율 리포트 제공 가능

## 카테고리 확장 순서

1. 소규모 업체를 위한 AX
2. 사무업무 자동화
3. 창업 문서 자동화
4. AI 보안과 운영
5. 바이브코딩
6. 제조업을 위한 AX

## 모듈 확장 순서

소규모 업체 카테고리:

1. 고객문의 자동응답과 상담 분류
2. 예약 문의를 일정으로 바꾸기
3. 리뷰 분석과 답변 자동화
4. 자동화된 상담 기록 관리
5. 사장님을 위한 주간 운영 보고서 만들기
6. 견적서와 안내문 빠르게 작성하기

사무업무 카테고리:

1. 회의 메모를 보고서로 바꾸기
2. 이메일 답변 기준 만들기
3. 반복 보고서 자동화
4. 자료 요약과 액션 아이템 정리

창업 문서 카테고리:

1. 아이디어 메모를 사업계획서 초안으로 바꾸기
2. 고객 문제 정의하기
3. 경쟁 서비스 분석하기
4. 발표자료 흐름 만들기

## 출시 후 피드백 루프

- 실습 시작률 확인
- 어느 단계에서 이탈하는지 확인
- 템플릿 다운로드율 확인
- 사용자가 실제로 어떤 답변 초안을 수정하는지 확인
- 파일럿 인터뷰 진행
- 가장 반응 좋은 실습을 유료화

## 핵심 판단

초기 제품 성공의 기준은 강의 수가 아닙니다. 단일 실습 모듈에서 사용자가 실제 업무에 가져갈 수 있는 결과물을 만들었는지입니다.

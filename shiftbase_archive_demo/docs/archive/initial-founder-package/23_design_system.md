# 디자인 시스템 초안

## 디자인 방향

Shiftbase는 친근하지만 전문적인 실습형 학습 플랫폼이어야 합니다.

디자인 키워드:

- 실무적
- 명확함
- 신뢰감
- 차분함
- 인터랙티브
- 어렵지 않음

## 시각 톤

- 배경은 Soft Slate와 흰색 중심
- 주요 액션은 Indigo Shift
- 보조 면과 부드러운 강조는 Sky Wing
- 강조 포인트는 Electric Violet
- 성공, 경고, 오류 색상은 상태 표현에만 제한적으로 사용


## 타이포그래피

권장:

- 한국어 가독성이 좋은 산세리프
- 웹 기본으로는 Pretendard 계열 또는 시스템 폰트

계층:

- H1: 48에서 64
- H2: 32에서 40
- H3: 24에서 28
- Body: 15에서 17
- Caption: 12에서 14

## 레이아웃 원칙

- 데스크톱 실습 화면 우선
- 12컬럼 그리드 기반
- 카드 간 여백 충분히 확보
- 오른쪽 패널은 상태와 다음 행동에 집중
- 하단 패널은 자료와 결과 정리에 사용

## 컴포넌트 스타일

### 카드

- 배경: 흰색
- 테두리: 연한 회색
- radius: 16에서 24
- shadow: 약함

### 버튼

Primary:

- 배경: primary
- 텍스트: 흰색
- 목적: 다음 단계, 시작, 저장

Secondary:

- 배경: 흰색
- 테두리: primary_light 또는 line
- 텍스트: primary
- 목적: 맛보기, 자세히 보기, 이전 단계

Ghost:

- 배경 없음
- 목적: 보조 액션

### 배지

용도:

- 카테고리
- 난이도
- 상태
- 진행 중
- 완료

### 프로그레스

- 상단 전체 진도
- 실습 단계별 진도
- 모듈 카드 진행률

### 스텝퍼

- 번호와 상태를 함께 표시
- 완료는 초록 체크
- 현재 단계는 파란 강조
- 대기는 회색

## 실습 화면 패턴

### 좌측 패널

역할:

- 현재 단계 위치 파악
- 완료 상태 확인
- 학습자가 길을 잃지 않게 안내

### 중앙 패널

역할:

- 실제 조작이 일어나는 메인 작업 공간
- 문의함, 문서, 예약표, 리뷰 목록 등 표시

### 우측 패널

역할:

- 분석 결과
- 체크리스트
- 점수
- 다음 단계 안내

### 하단 패널

역할:

- 워크플로우 요약
- 자료와 도구
- 결과 리포트

## 아이콘 스타일

- 라인 아이콘 중심
- 과도한 장식 지양
- 기능별 색상 제한
- 아이콘만으로 의미 전달하지 말고 텍스트 병기

## 상태 표현

완료:

- 초록 체크
- 완료 텍스트

진행 중:

- 파란 강조
- 진행 중 텍스트

대기:

- 회색
- 대기 텍스트

오류:

- 붉은색 아이콘
- 구체적 설명

## 빈 상태 문구

진행 중인 실습이 없습니다.
새로운 모듈을 선택해 첫 업무전환 실습을 시작해보세요.

저장된 결과물이 없습니다.
실습을 완료하면 결과 리포트와 템플릿이 이곳에 저장됩니다.

## 마이크로카피 원칙

- 행동을 명확히 지시
- 사용자를 탓하지 않음
- AI 결과는 초안이라고 표현
- 다음에 무엇을 해야 하는지 안내

좋은 예:

- 선택한 문의의 유형을 골라주세요.
- 답변에 필요한 정보가 빠졌는지 확인해보세요.
- AI가 만든 초안입니다. 발송 전 직접 검토하세요.

나쁜 예:

- 잘못된 입력입니다.
- 자동화 완료.
- AI가 정답을 만들었습니다.

## 핵심 판단

Shiftbase의 디자인은 강의 사이트보다 업무 도구에 가까워야 합니다. 그러나 너무 복잡한 SaaS처럼 보이면 초보자가 부담을 느낍니다. 학습 플랫폼과 업무 도구 사이의 균형이 중요합니다.

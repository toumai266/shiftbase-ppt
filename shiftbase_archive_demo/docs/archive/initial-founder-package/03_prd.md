# PRD

## 제품명

Shiftbase

## 문서 목적

이 문서는 Shiftbase의 초기 제품 요구사항을 정의합니다. 초기 제품은 영상 중심 LMS가 아니라, 웹사이트 자체가 교육 인터페이스가 되는 실습형 학습 플랫폼입니다.

## 제품 목표

Shiftbase는 사용자가 실제 업무 상황을 웹 인터페이스로 따라 하며 AI 업무전환 방법을 배우도록 돕습니다.

초기 목표는 다음과 같습니다.

- 소규모 업체와 작은 팀을 위한 고객문의 자동응답 학습 모듈 출시
- 사용자가 실습을 완료하고 결과물을 저장하게 만들기
- 템플릿과 체크리스트를 통해 현업 적용 가능성을 높이기
- AI 기능 없이도 학습 가능한 인터페이스를 만들고, 이후 AI 보조 기능을 추가하기
- B2C 파일럿과 B2B 파일럿 모두 가능한 구조 확보

## 제품 비목표

초기 제품에서 하지 않을 일은 다음과 같습니다.

- 영상 강의 플랫폼 구현
- 전체 LMS 기능 구현
- 강사 마켓플레이스 구현
- 모든 업종 카테고리 동시 출시
- 복잡한 구독 요금제 도입
- 완전 자동화 SaaS 제공
- 사용자의 실제 고객 데이터 대규모 수집

## 주요 사용자

### 소규모 업체 운영자

고객문의, 예약, 리뷰, 매장 운영, 정산, 문서 업무를 처리합니다. AI를 쓰고 싶지만 어디서부터 시작해야 할지 모릅니다.

### 작은 팀의 실무자

고객 응대, 사무 처리, 보고서 작성, 회의록 정리 같은 반복 업무가 많습니다. 도구는 사용할 수 있지만 업무흐름 재설계 경험은 부족합니다.

### 1인 창업자

사업계획서, 홍보글, 문의 응대, 제안서, 견적서 등을 직접 처리합니다. 빠르게 업무 체계를 만들고 싶어합니다.

### 기업교육 담당자

직원들이 AI를 실제 업무에 적용하길 원합니다. 단순 영상 시청이 아니라 실습 결과와 진도 리포트를 원합니다.

## 핵심 사용자 여정

### 첫 방문자

1. 홈에서 Shiftbase의 차별점을 이해한다.
2. 소규모 업체를 위한 AX 카테고리를 클릭한다.
3. 고객문의 자동응답과 상담 분류 모듈을 확인한다.
4. 무료 실습을 시작한다.
5. 실습 결과 리포트와 템플릿을 받는다.
6. 추가 모듈 또는 유료 패키지에 관심을 가진다.

### 학습자

1. 로그인한다.
2. 나의 학습 대시보드에서 진행 중 모듈을 확인한다.
3. 실습실에서 고객 문의를 분류한다.
4. 답변 초안을 생성하거나 선택한다.
5. 체크리스트로 답변을 검토한다.
6. 결과를 저장한다.
7. 템플릿을 다운로드한다.
8. 다음 모듈로 이동한다.

### 기업교육 관리자

초기 MVP에서는 완성하지 않지만 설계에 반영합니다.

1. 조직 계정을 만든다.
2. 구성원에게 수강권을 부여한다.
3. 구성원 진도와 실습 결과를 확인한다.
4. 수료 기준 충족 여부를 확인한다.
5. 팀별 업무 적용 리포트를 받는다.

## 핵심 기능 요구사항

### 홈 페이지

필수 요소:

- 브랜드명과 슬로건 표시
- 플랫폼의 차별점 설명
- 카테고리 카드
- 추천 학습 모듈
- 실습형 학습 구성 안내
- 로드맵 섹션
- 회원가입 또는 실습 시작 CTA

성공 기준:

- 사용자가 영상 강의 플랫폼이 아니라 실습형 학습 제품임을 이해해야 함
- 소규모 업체와 일반 실무자가 자신의 문제와 연결해야 함

### 카테고리 페이지

필수 요소:

- 카테고리 설명
- 대표 모듈
- 하위 업무 영역 카드
- 추천 학습 모듈
- 카테고리 로드맵
- 제공되는 실무 자산

초기 카테고리:

- 소규모 업체를 위한 AX

추후 카테고리:

- 사무업무 자동화
- 창업 문서 자동화
- AI 보안과 운영
- 바이브코딩
- 제조업을 위한 AX

### 모듈 상세 페이지

필수 요소:

- 모듈 개요
- 학습 목표
- 실습 결과물
- 커리큘럼
- 포함 자료
- 실무 적용 사례
- 수강 시작 버튼

초기 모듈:

- 고객문의 자동응답과 상담 분류

### 실습실 페이지

필수 요소:

- 실습 단계 표시
- 가상 업무 화면
- 사용자가 수행할 액션
- AI 적용 포인트
- 힌트
- 검증 결과
- 진행률
- 다음 단계 이동
- 실습 자료와 도구
- 결과 리포트

초기 실습 시나리오:

- 고객 문의 워크플로우 실습실

### 나의 학습 대시보드

초기 필수 요소:

- 진행 중 모듈
- 완료한 실습
- 저장한 결과물
- 다운로드 가능한 템플릿
- 최근 활동

추후 요소:

- 수료증
- 조직 소속 표시
- 팀 리포트

## 실습 엔진 요구사항

실습 엔진은 다음을 처리해야 합니다.

- 단계 렌더링
- 클릭, 선택, 입력 액션 처리
- 사용자 상태 저장
- 정답 또는 적합성 검증
- 힌트 제공
- 점수 또는 완료 여부 계산
- 결과 리포트 생성
- 템플릿 다운로드 연결

## AI 기능 요구사항

초기에는 AI 없이도 작동해야 합니다.

1차 AI 기능:

- 고객문의 유형 분류
- 답변 초안 생성
- 답변 품질 평가
- 누락 정보 안내
- 현업 적용 힌트 제공

AI 사용 원칙:

- 민감정보 입력 최소화
- 예시 데이터 우선 사용
- 사용자가 직접 검토하고 수정하는 구조
- AI 결과를 정답으로 단정하지 않음

### P0

- 홈 페이지
- 카테고리 페이지
- 모듈 상세 페이지
- 실습실 단일 모듈
- 로그인
- 진도 저장
- 실습 결과 저장
- 템플릿 다운로드

### P1

- AI 분류 기능
- 답변 초안 생성
- 결과 리포트
- 나의 학습 대시보드
- 유료 수강권
- 결제

### P2

- 기업교육 조직 관리
- 관리자 페이지
- 수료증
- 학습 분석 리포트
- 콘텐츠 저작 도구

## 비기능 요구사항

- 주요 페이지 초기 로딩 속도는 빠르게 유지
- 모바일에서도 최소한 학습 진행과 자료 다운로드 가능
- 실습실은 데스크톱 우선 최적화
- 사용자 입력값은 안전하게 저장
- 조직 데이터는 분리 관리
- AI 호출 실패 시 대체 예시 결과 제공
- 실습 상태는 중간 저장 가능

## 성공 지표

초기 핵심 지표:

- 무료 실습 시작률
- 실습 완료율
- 실습 결과 저장률
- 템플릿 다운로드율
- 다음 모듈 이동률
- 이메일 또는 회원가입 전환율
- 유료 구매 의향 응답률

제품 품질 지표:

- 평균 실습 시간
- 힌트 사용률
- 중도 이탈 단계
- AI 결과 수정률
- 체크리스트 통과율

## 출시 기준

MVP 출시 조건:

- 홈, 카테고리, 모듈 상세, 실습실이 연결됨
- 고객 문의 실습을 처음부터 끝까지 완료할 수 있음
- 실습 결과가 저장됨
- 템플릿 다운로드가 가능함
- 주요 오류 없이 데모 가능함
- 사용자 10명 이상에게 파일럿 테스트 가능함

## 핵심 제품 문장

Shiftbase는 영상을 보는 플랫폼이 아니라, 업무 화면을 따라 하며 AI 적용법을 익히는 웹 기반 학습 인터페이스입니다.

# AI 비서 컨테이너 현재 상태 보고

작성일: 2026-06-05  
대상: `content/containers/ai-personal-assistant-after-hours/container.json`

제외 범위: 코드 오류, 버그, 보안 구현, 데모 데이터 정리, 폴백 정리

## 1. 이 문서의 판단

`퇴근 후 1시간, 나만의 AI 비서 만들기` 컨테이너는 현재 Shiftbase의 대표 간판이 될 만한 주제와 욕망을 갖고 있다. 제목은 직관적이고, 사용자가 "나도 내 AI 비서를 만들 수 있나?"라고 느끼게 할 힘이 있다.

하지만 현재 본문과 워크스페이스는 아직 그 약속을 충분히 증명하지 못한다. 파일 상태는 `published`지만, 실제 읽기 경험 기준으로는 대표 컨테이너 후보 또는 개편 전 공개본에 가깝다.

가장 큰 문제는 내용이 부족한 것이 아니다. 오히려 구조와 항목은 많다. 문제는 사용자가 읽었을 때 바로 쓸 수 있는 장면보다 설명, 원칙, 체크리스트, 매뉴얼이 앞선다는 점이다. 인터랙티브도 대부분 사용자의 판단을 바꾸는 장치라기보다, 이미지, 예시 목록, 체크리스트, 빈 결과 카드로 배치되어 있다.

사용자가 최근 지적한 문제, 즉 "AI가 만든 내용이 사람이 읽었을 때 쓸모없고, 읽기 어렵고, 표현이 명확하지 않고, 인터랙티브가 쓸모없다"는 감각은 이 컨테이너에도 직접 적용된다.

즉, 현재 파일 구조만 보면 이 컨테이너는 인터랙션이 풍부한 도구형 경험이 아니다. `31개` 워크스페이스 블록 중 `30개`는 정적 블록이고, 실제 실행형 블록은 `1개`다.

## 3. 현재 컨테이너가 약속하는 경험

이 컨테이너는 표면적으로 다음 경험을 약속한다.

- 퇴근 후 한 시간 안에 AI 비서의 첫 버전을 만든다.
- Telegram 또는 Discord에서 AI 비서를 부른다.
- Hermes 또는 OpenClaw 기반 연결 흐름을 이해한다.
- Gmail과 Calendar를 읽기 중심으로 연결해 아침 브리핑을 만든다.
- 자동 실행보다 사람 승인 기준을 먼저 세운다.
- 마지막에 AI 비서 운영 규칙과 7일, 14일 루틴을 남긴다.

이 약속은 후킹 자체는 강하다. 사용자가 원하는 것은 "AI 비서 개념 설명"이 아니라 실제로 내 메일, 일정, 메모, 할 일을 정리해주는 느낌이다. 제목과 추천 카드도 그 욕망을 향하고 있다.

하지만 현재 페이지를 읽으면 경험의 중심이 "비서를 만들어서 써본다"보다 "비서를 안전하게 설계하는 원칙을 배운다" 쪽으로 많이 기운다. 이 차이가 크다.

## 4. 실제 페이지에서 보이는 경험

현재 페이지 흐름은 크게 세 덩어리다.

첫째, 역할과 경계 설정이다. `module-01`은 비서에게 맡길 일, 맡기면 안 되는 일, 역할 선언문을 다룬다. 내용은 맞지만, 첫 화면부터 사용자가 실제로 보는 장면은 약하다. "갖고 싶은 것은 봇이 아니라 운영 능력", "욕망을 일감으로 바꾸기" 같은 표현은 설명으로는 맞지만 즉시 행동을 만들지는 않는다.

둘째, 브리핑 형식 설계다. `module-02`는 아침 브리핑, 퇴근 브리핑, 일일 브리핑 설계표를 다룬다. 이 부분은 현재 컨테이너에서 가장 제품 방향과 잘 맞는다. 사용자가 AI 비서에게 받고 싶은 결과물이 짧은 브리핑이라는 점이 명확하기 때문이다. 다만 여전히 "좋은 브리핑은..."이라는 설명이 많고, 실제 사용자의 하루 자료가 브리핑으로 바뀌는 장면은 부족하다.

셋째, 도구 연결과 운영 규칙이다. `module-03`부터 `module-06`은 Telegram, Discord, Gmail, Google Calendar, OAuth, token, pairing, allowlist, 7일 루틴, 14일 루틴을 다룬다. 이 구간은 구체 도구명이 많아 실제성이 있어 보이지만, 비전공자에게는 곧바로 매뉴얼 더미처럼 느껴질 수 있다. 특히 Discord 연결 매뉴얼과 Telegram 연결 매뉴얼은 본문 길이가 가장 긴 편이고, 사용자는 "비서 만들기"보다 "설정 문서를 읽는 중"이라고 느낄 수 있다.

## 5. 사람이 읽는 콘텐츠로서의 현재 약점

### 5-1. 첫 장면이 약하다

현재 첫 페이지는 "AI 비서를 만들고 싶다는 욕망은 기술 호기심에서 끝나지 않는다"는 설명으로 시작한다. 하지만 사용자가 바로 상상할 수 있는 구체 장면은 늦게 나온다.

예를 들어 독자가 기대하는 첫 장면은 이런 쪽에 가깝다.

```text
아침 8시 10분, Telegram에 오늘 먼저 볼 일 3개가 도착했다.
10시 회의 전에 확인할 파일, 답장할 메일, 어제 미룬 일이 한 화면에 잡힌다.
```

현재 컨테이너는 이런 장면보다 "운영 능력", "작은 운영자", "정리 체계" 같은 설명어를 먼저 보여준다. 그래서 제목은 직관적인데 본문은 바로 체감되지 않는다.

### 5-2. 원칙은 많지만 사용자의 일이 잘 보이지 않는다

컨테이너 전체에 반복되는 메시지는 분명하다.

- 자동 전송 금지
- 읽기, 요약, 초안, 질문 제안까지만 허용
- 원문 외부 전송 금지
- pairing 또는 allowlist
- token 회수
- Gmail과 Calendar는 읽기 중심

이 원칙들은 중요하다. 다만 현재 형태에서는 사용자의 실제 하루, 실제 메일, 실제 일정, 실제 메모가 충분히 보이지 않는다. 결과적으로 안전 기준은 알겠는데, 사용자가 "내 일이 줄었다"는 감각은 약하다.

### 5-3. 인터랙티브 대부분이 판단을 바꾸지 않는다

현재 워크스페이스는 많지만, 사용자의 선택이 콘텐츠 결과를 바꾸는 장면은 거의 없다.

`image-definition`은 참고 이미지다.  
`example-list`는 읽는 목록이다.  
`checklist`는 확인 항목이다.  
`result-card`는 사용자가 나중에 채워야 할 빈 자리다.  
`code-canvas`는 하나뿐이고, 그마저 버튼을 누르면 고정된 5줄 브리핑이 출력되는 구조다. 내부 구현도 사용자의 입력이나 선택을 반영하는 흐름이 아니라, 버튼 클릭 시 미리 작성된 문장을 `innerHTML`로 넣는 방식이다.

따라서 현재 인터랙티브는 "읽기 부담을 줄이는 도구"라기보다 "우측에 배치된 보조 요소"에 가깝다. 사용자가 말한 "쓸모 없는 인터랙티브"라는 문제의식은 여기서 나온다.

### 5-4. `퇴근 후 1시간` 약속과 실제 분량 사이에 긴장이 있다

제목은 한 시간 안에 AI 비서를 만드는 기대를 만든다. 하지만 실제 컨테이너는 22페이지이고, 다루는 범위도 Hermes, OpenClaw, Telegram, Discord, Gmail API, Calendar API, Google Cloud, OAuth, BotFather, Developer Portal, token, intent, allowlist, pairing까지 넓다.

비전공자나 초급 사용자는 이 범위를 보고 "한 시간 안에 된다"보다 "생각보다 복잡하다"고 느낄 수 있다. 현재 컨테이너는 한 시간짜리 실행 경험이라기보다, AI 비서 운영 설계와 연결 준비를 넓게 훑는 초안에 가깝다.

### 5-5. 실패 장면이 설명으로만 존재한다

컨테이너는 실패를 경고한다. token 노출, 원문 외부 전송, 자동 발송, 삭제, 결제, 고객 연락 같은 위험을 반복해서 말한다.

하지만 실제 페이지 안에서 AI가 잘못된 브리핑을 만들고, 사용자가 어디서 멈추고 고치는지 보여주는 장면은 약하다. "이렇게 하면 위험하다"는 설명은 있지만, 독자가 위험한 AI 초안과 안전한 최종안을 비교하며 판단하는 경험은 부족하다.

Shiftbase가 말하려는 가치가 "AI 결과를 사람이 판단하고 고친다"라면, 현재 AI 비서 컨테이너는 그 가치를 설명하고 있을 뿐 충분히 경험시키지는 못한다.

## 6. 현재 가치

이 컨테이너의 가치는 세 가지다.

첫째, 주제의 후킹이 좋다. `AI 비서 만들기`는 사용자가 쉽게 욕망을 느끼는 주제다. 메일, 일정, 메모, 할 일, Telegram, Discord 같은 소재도 일상과 가깝다.

둘째, 안전선이 비교적 분명하다. 자동 전송 금지, 읽기 중심, 사람 승인, token 회수, 원문 외부 전송 금지 같은 기준은 컨테이너 전반에 반복된다. 이 방향은 Shiftbase가 단순 자동화 튜토리얼과 달라질 수 있는 지점이다.

셋째, 대표 컨테이너로 삼을 잠재력이 있다. 현재 공개 컨테이너 중에서는 "내가 뭔가 만들 수 있다"는 느낌을 가장 잘 줄 수 있는 제목과 범위를 갖고 있다.

하지만 현재 상용 콘텐츠로서의 읽기 가치는 아직 약하다. 구조가 많고 주제가 넓지만, 한 페이지를 읽었을 때 "이건 내 일이다", "이 문장은 바로 쓸 수 있다", "이 버튼을 눌러 내 판단이 바뀌었다"는 감각이 충분하지 않다.

## 7. 현재 상태를 한 문장으로 요약

`AI 비서 만들기` 컨테이너는 대표 간판 후보로는 맞지만, 현재 본문은 사람이 읽고 쓰는 경험보다 AI가 만든 설명서와 체크리스트의 밀도가 더 강하다.

따라서 지금 상태는 "런칭된 완성본"이라기보다, 좋은 제목과 좋은 범위를 가진 공개 초안으로 보는 편이 정확하다.

# AI 웹북 전체 콘텐츠 피드백: 인터랙티브 강박 금지

작성일: 2026-06-01  
적용 범위: `content/containers`의 모든 AI 웹북 컨테이너, 모듈, 페이지  
상태: 전체 콘텐츠 저작 원칙  
2026-06-02 업데이트: 전략 피드백 반영

## 1. 핵심 피드백

모든 도구와 모든 페이지를 반드시 인터랙티브하게 구성할 필요는 없다.

Shiftbase의 차별점은 "움직이는 화면이 많다"가 아니라, 독자가 실제 업무 장면을 더 잘 이해하고 더 잘 판단하게 되는 경험이다. 따라서 어떤 페이지는 단순 매뉴얼, 단계별 안내, 참고 이미지, 비교표, 체크리스트, 예시 문장만으로도 충분하다.

인터랙티브는 콘텐츠 품질을 높이는 수단일 수 있지만 목표가 아니다.

2026-06-02 전략 피드백 이후의 추가 원칙은 다음이다.

- 콘텐츠 형식은 `무엇을 만들기 쉬운가`가 아니라 `어떤 구매 이유를 증명하는가`로 선택한다.
- 6월 유료 검증의 무료 진단 훅은 `위험한 AI 초안 5분 진단`으로 둔다.
- `퇴근 후 1시간, 나만의 AI 비서 만들기`는 필요한 욕망형 유입·확장 컨테이너다. 다만 단순 봇 만들기가 아니라 권한 분리, 사람 승인, 개인정보 제외, 실패 시 멈춤 조건을 포함한 판단형 AI 비서 설계 경험으로 구성한다.
- 2026년 6월의 유료 검증은 `ai-email-document-communication`과 `ai-report-writing-dashboard`를 우선한다.
- 다른 컨테이너의 콘텐츠 개선은 중단이 아니라 판매 전면 노출 보류로 해석한다.

## 2. 새 판단 기준

페이지를 만들 때 먼저 물어야 할 질문은 "무엇을 조작하게 할까?"가 아니다.

먼저 아래 순서로 판단한다.

1. 이 페이지에서 독자가 이해해야 할 핵심 판단은 무엇인가.
2. 그 판단을 가장 쉽게 보여주는 형식은 무엇인가.
3. 조작이 실제 이해를 높이는가, 아니면 화면만 복잡하게 만드는가.
4. 정적 매뉴얼이나 참고 이미지가 더 빠르고 선명한가.
5. 산출물은 독자의 관점 변화를 증명하는가.

## 3. 형식 선택 원칙

| 형식 | 적절한 경우 | 피해야 할 경우 |
| --- | --- | --- |
| 단순 매뉴얼 | 설치 전 준비, 계정 권한, 단계별 확인, 운영 기준을 설명할 때 | 독자가 직접 비교하거나 분류해야 하는 장면 |
| 참고 이미지 | 화면 구조, 설정 위치, 업무 흐름, 결과물 예시를 빠르게 보여줄 때 | 이미지가 장식에 그치거나 본문과 같은 말을 반복할 때 |
| 비교표 | 선택 기준, 도구 차이, 금지/허용 범위, before/after를 보여줄 때 | 표가 너무 많아 독서 리듬을 끊을 때 |
| 체크리스트 | 발송 전 검토, 개인정보 확인, 승인선 점검처럼 누락 방지가 중요할 때 | 체크 항목이 일반론으로 반복될 때 |
| 인터랙티브 | 선택, 분류, 입력, 비교, 수정, 저장이 실제 학습 행동을 만들 때 | 단순 설명을 억지로 버튼과 슬라이더로 바꿀 때 |
| code-canvas | 도메인별 작은 도구, 시뮬레이터, 입력 기반 결과 확인이 필요할 때 | 정적 가이드나 예시 이미지로 충분한 페이지 |

## 4. 전체 콘텐츠 적용 규칙

- 인터랙티브 수, code-canvas 수, 클릭 요소 수를 품질 근거로 쓰지 않는다.
- 모든 모듈에 같은 패턴의 프롬프트 랩, 점수판, 버튼 UI를 반복하지 않는다.
- 우측 워크스페이스는 실행형 도구일 수도 있고, 단순 참고 이미지나 매뉴얼 카드일 수도 있다.
- 초보자용 무료 콘텐츠는 특히 조작보다 이해의 속도와 심리적 진입 장벽을 우선한다.
- 유료 콘텐츠도 조작이 필요하지 않은 설명 페이지에서는 정적 자료를 허용한다.
- 좋은 매뉴얼 페이지는 나쁜 인터랙티브 페이지보다 낫다.

## 5. 통과 예시

### 단순 매뉴얼이 더 좋은 장면

- Telegram 또는 Discord에서 AI 비서를 부르기 전 권한을 나누는 기준
- 이메일 자동 보고를 시작하기 전 원문 제외, 발송 금지, 보고 시간 제한을 설명하는 페이지
- 개인정보를 AI에 넣기 전 금지 정보와 비식별 예시를 보여주는 페이지
- 보고서 첫 문장을 고치는 before/after 문장 비교

### 인터랙티브가 필요한 장면

- 여러 회의 발언을 결정, 의견, 보류, 액션으로 분류하는 페이지
- 엑셀 표의 열 의미와 오류 후보를 사용자가 골라보는 페이지
- 고객 문의를 긴급도와 에스컬레이션 기준으로 나눠보는 페이지
- 사용자가 자기 업무 자료를 넣고 최종 공유 문장을 만드는 페이지

## 6. 금지되는 오해

다음 해석은 금지한다.

- "인터랙티브가 많으면 유료 품질이 올라간다."
- "모든 도구는 조작 가능한 캔버스로 만들어야 한다."
- "정적 이미지나 매뉴얼은 품질이 낮은 대체재다."
- "우측 워크스페이스가 비어 보이지 않게 무엇이든 움직이게 해야 한다."

올바른 해석은 다음이다.

> 형식은 콘텐츠 판단을 돕기 위한 선택이다. 독자가 더 빨리 이해하고 더 안전하게 실행한다면, 단순 매뉴얼과 참고 이미지도 충분히 좋은 콘텐츠다.

## 7. 작업 지시

앞으로 모든 컨테이너 개선 작업에서 각 페이지는 먼저 `매뉴얼`, `참고 이미지`, `비교표`, `체크리스트`, `인터랙티브`, `code-canvas` 중 가장 낮은 복잡도의 형식을 선택한다.

복잡한 인터랙티브를 만들려면 다음 조건을 만족해야 한다.

1. 사용자가 실제로 선택, 입력, 분류, 수정, 저장 중 하나를 해야 한다.
2. 그 행동이 모듈의 핵심 판단을 강화한다.
3. 정적 매뉴얼이나 참고 이미지보다 이해가 쉬워진다.
4. 결과가 다음 페이지나 최종 산출물에 의미 있게 남는다.

이 조건을 만족하지 못하면 단순 매뉴얼, 참고 이미지, 비교표, 예시 문장으로 구성한다.

## 8. 전략 적합성 체크

각 페이지와 모듈을 개선하기 전에 다음 질문을 추가로 확인한다.

1. 이 콘텐츠는 `AI 초안을 사람이 고치는 판단력`이라는 승리 이론을 강화하는가.
2. 독자가 이 장면을 보고 "내 업무에서도 돈 낼 문제"라고 느낄 수 있는가.
3. 무료 샘플이라면 유료 상품을 살 사람을 선별하는가, 아니면 자동화/도구 연결에 관심 있는 사람만 모으는가.
4. 산출물이나 인터랙티브가 아니라 자기 업무 적용, 수정 이유 설명, 최종 공유 문장으로 학습 결과가 남는가.
5. 이 모듈이 2026년 6월 유료 검증 대상 2개 컨테이너에 직접 기여하는가.

5번에 `아니오`라면 삭제하지 않는다. 다만 6월 판매·홍보 우선순위에서는 뒤로 미룬다.

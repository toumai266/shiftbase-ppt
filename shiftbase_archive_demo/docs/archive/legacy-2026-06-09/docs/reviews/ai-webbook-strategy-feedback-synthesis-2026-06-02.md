# AI 웹북 전략 피드백 반영 메모

작성일: 2026-06-02  
대상: Shiftbase AI 웹북 사업전략, 런칭 계획, 콘텐츠 개선 기준  
상태: 전략 재정렬 기준

## 1. 근거와 한계

이 문서는 2026-06-02에 받은 외부 모델 피드백과 유튜브 영상 `진짜 '사업 전략'은 이래야 합니다`의 한국어 자막 328개 구간을 바탕으로 작성한 실행 기준이다.

주의할 점은 다음이다.

- 이 문서는 시장성, 법률, 재무 타당성 검증 보고서가 아니다.
- ChatGPT Pro 응답은 자문성 피드백이며, 독립 검증된 리서치가 아니다.
- 유튜브 영상은 전략 프레임을 적용하기 위한 참고 자료이며, Shiftbase의 성공 가능성을 증명하지 않는다.
- Kodak, Instagram, Southwest, IKEA, Musinsa 사례는 개념 설명용 예시로만 다룬다.

## 2. 전략 프레임

영상의 핵심은 다음 세 문장으로 정리된다.

1. 행동 목록의 합은 전략이 아니다.
2. 전략은 선택한 경기장에서 어떻게 이길지에 대한 일관된 승리 이론이다.
3. 좋은 전략은 `where to play`, `how to win`, 필요한 역량, 관리 시스템, `what must be true` 가정으로 한 장에 정리될 수 있어야 한다.

따라서 Shiftbase의 다음 작업은 더 많은 콘텐츠를 만드는 것이 아니라, 구매자가 왜 다른 AI 교육 대신 이 웹북을 선택해야 하는지 증명하는 일이다.

## 3. 새 전략 결론

2026년 6월의 목표는 공개 런칭이 아니라 유료 검증이다.

현재 계획은 `컨테이너 개선`, `무료 상품`, `런칭 일정`, `마케팅 채널`이 잘 정리되어 있지만, 그대로 두면 실행 목록이 전략처럼 보이는 위험이 있다. 이제 기준을 다음처럼 바꾼다.

- 6개 유료 후보를 모두 팔지 않는다.
- 유료 검증 대상은 `ai-email-document-communication`과 `ai-report-writing-dashboard` 2개로 줄인다.
- `ai-meeting-action-system`, `ai-spreadsheet-analysis-automation`, `ai-customer-support-playbook`, `ai-risk-privacy-governance`는 상품 후보나 B2B 신뢰 자산으로 보류한다.
- 6월 유료 검증의 무료 진단 훅은 `위험한 AI 초안 5분 진단`으로 둔다.
- `퇴근 후 1시간, 나만의 AI 비서 만들기`는 필요한 욕망형 컨테이너다. 다만 6월의 핵심 유료 검증 2개를 대체하지 않고, 메일/문서와 보고서/대시보드 컨테이너로 이어지는 브리지로 둔다.
- 공개 런칭 일정은 유지하더라도, 그 전제는 실제 결제와 자기 업무 적용 사례다.

## 4. 한 장 전략

| 항목 | 결정 |
| --- | --- |
| Where to play | AI 초안을 이미 업무에 쓰지만 그대로 보내기 불안한 PM, 운영, 마케팅, CS 리드, 주니어 매니저 |
| How to win | 프롬프트나 자동화가 아니라, 위험한 AI 초안을 업무 문장으로 고치는 판단력을 훈련시킨다 |
| 첫 유료 상품 | `ai-email-document-communication`, `ai-report-writing-dashboard` |
| 메인 무료 훅 | `위험한 AI 초안 5분 진단` |
| 욕망형 브리지 | `퇴근 후 1시간, 나만의 AI 비서 만들기` |
| 필요한 역량 | 실제 업무 원문 같은 예시, 위험한 AI 초안 설계, 수정 판단 해설, 자기 업무 적용 루브릭 |
| 관리 시스템 | 완주율보다 자기 업무 적용률, 수정 이유 설명, 익명 전후 사례, 실제 결제 전환을 추적 |
| What must be true | 사용자가 단순 AI 사용법보다 AI 초안 검수 능력에 돈을 낼 만큼 고통을 느껴야 한다 |

## 5. 2주 검증 기준

2026-06-02부터 2026-06-15까지는 아래 기준으로 Go/No-Go를 판단한다.

| 지표 | 통과 기준 |
| --- | --- |
| 무료 샘플 시작자 | 30명 이상 |
| 무료 샘플 완료자 | 18명 이상 |
| 자기 업무 적용 제출 | 9명 이상 |
| 실제 결제 | 39,000원 이상 5건 |
| 팀 파일럿 내부 공유 | 3건 이상 |
| 공개 가능한 익명 사례 | 3건 이상 |
| 문의 대응 시간 | 1인당 20분 이하 |

말로만 "살 의향이 있다"는 응답은 통과 증거가 아니다. 결제, 세금계산서 발행 정보, 내부 공유, 자기 업무 적용 결과처럼 행동이 남아야 한다.

## 6. 가격과 패키지 기준

| 상품 | 기준 가격 | 원칙 |
| --- | --- | --- |
| 개인 유료 베타 1개 컨테이너 | 39,000원 | 첫 50명 한정, 피드백 제공 조건 |
| 개인 유료 베타 2개 묶음 | 69,000원 | 메일/문서 + 보고서 중심 |
| 공개 런칭 1개 컨테이너 | 59,000원 | 베타 검증 후 적용 |
| 공개 런칭 2개 묶음 | 99,000원 | 핵심 패키지 |
| 팀 셀프서브 패키지 | 490,000원 | 10명 이하, 커스터마이징 없음 |
| 진행형 팀 파일럿 | 1,500,000원~3,000,000원 | 90분 킥오프, 2주 학습, 결과물 리뷰 1회 |
| B2B 커스텀 | 5,000,000원부터 | 내부 문서 반영, 보안 검토, 직무별 예시 포함 |

19,000원 가격은 기본 가설에서 제거한다. 콘텐츠 품질을 최상위 기준으로 잡는다면 너무 낮은 가격은 PDF나 프롬프트 묶음처럼 보이는 신호가 된다.

## 7. 마케팅 수정 기준

버릴 메시지:

- `AI로 업무 자동화`
- `생산성 2배`
- `초보자도 하루 만에`
- 프롬프트, 템플릿, 산출물 수 강조
- Hermes, OpenClaw 같은 도구명 중심 메시지

살리되 수정할 메시지:

- `퇴근 후 1시간, 나만의 AI 비서 만들기`
- 단, "봇 만들기"가 아니라 "내 일을 정리하고 다음 행동을 알려주는 개인 운영체계"로 포지셔닝한다.
- Telegram, Discord, 이메일 연동은 욕망을 보여주는 장치이며, 권한 분리와 사람 승인 기준을 반드시 포함한다.

살릴 메시지:

- AI가 쓴 초안, 사람이 고친 판단
- 위험한 AI 초안을 그대로 보내기 전에 고치는 기준
- 메일과 보고서에서 AI 결과를 어디까지 믿고 어디서 멈출지 배우는 웹북
- 내 업무 원문에 적용해 최종 공유 문장까지 만드는 훈련

채널은 넓게 뿌리는 방식보다 전후 비교 사례, 직무별 1페이지 랜딩, 목표 고객 직접 아웃리치, 파일럿 결과물 기반 케이스 스터디에 집중한다.

## 8. 콘텐츠 개선 기준 변경

콘텐츠 개선은 `모듈을 더 많이 고친다`가 아니라 `첫 유료 상품이 구매 이유를 증명한다`로 해석한다.

우선순위:

1. `ai-email-document-communication`
2. `ai-report-writing-dashboard`
3. 두 컨테이너의 무료 진단 샘플
4. 자기 업무 적용 루브릭
5. 익명 전후 사례 수집 구조
6. B2B 신뢰 문서로서 `ai-risk-privacy-governance` 일부 발췌

나머지 컨테이너는 삭제하지 않는다. 다만 2026년 6월에는 판매 전면에 올리지 않는다.

## 9. 즉시 결정

1. 기존 `퇴근 후 1시간, 나만의 AI 비서 만들기`는 필요한 욕망형 컨테이너로 유지하되, 6월 핵심 유료 검증 2개를 대체하지 않는다.
2. 6월 유료 검증의 무료 진단 훅은 `위험한 AI 초안 5분 진단`으로 기획한다.
3. 유료 검증 상품은 메일/문서와 보고서/대시보드 2개로 줄인다.
4. 6월 15일까지 결제 5건과 자기 업무 적용 9건이 없으면 공개 런칭 일정을 늦춘다.
5. 콘텐츠 추가 제작보다 실제 사용자 결과물과 익명 사례 확보를 우선한다.

# Project Terms

이 문서는 Shiftbase에서 사용하는 핵심 용어를 정의합니다.

## Shiftbase

도서형 설명글과 웹 기반 인터랙션을 결합한 페이지형 학습 플랫폼입니다.

Shiftbase의 학습자는 긴 글을 스크롤로 읽는 대신, 한 화면씩 넘기며 좌측의 설명을 읽고 우측의 워크스페이스에서 직접 조작합니다.

## Hub

큰 주제 또는 도구 단위의 상위 분류입니다.

예시:

- AI 업무 활용
- Codex
- ChatGPT
- AI 데이터 분석

허브는 내부 분류와 탐색 구조에 가깝습니다. 사용자가 실제로 선택해서 학습을 시작하는 단위는 아닙니다.

## Track

허브 안에서 학습 목적이나 사용자 의도에 따라 나뉘는 경로입니다.

예시:

- Codex로 개발하기
- Codex로 업무 자동화하기
- Codex 에이전트 활용

트랙은 추천 경로, 필터, 섹션 구분에 사용합니다. 트랙 자체를 카드형 학습 상품처럼 다루지 않습니다.

## Container

사용자가 실제로 선택해서 들어가는 독립 학습 패키지입니다.

컨테이너는 단순한 강의가 아닙니다. 하나의 학습 목표를 달성하기 위해 본문, 시각화, 인터랙션, 실습, 리소스, 산출물을 담은 짧은 인터랙티브 학습서입니다.

컨테이너가 되려면 다음 조건을 만족해야 합니다.

- 대상자가 명확해야 합니다.
- 사용 상황이 명확해야 합니다.
- 끝났을 때 남는 산출물이 있어야 합니다.
- 카드 제목만 봐도 들어야 하는 이유를 판단할 수 있어야 합니다.
- 허브나 트랙보다 좁고, 레슨보다 큰 독립 단위여야 합니다.

## Module

컨테이너 안의 큰 장입니다.

예시:

- 내 업무 찾기
- AI 적용 지점 찾기
- 검토 기준 세우기

모듈은 학습 흐름을 나누는 구조적 단위이며, 사용자의 선택 카드로 노출되는 기본 단위는 아닙니다.

## Lesson

모듈 안에서 하나의 작은 학습 흐름을 담당하는 단위입니다.

레슨은 여러 페이지 또는 스텝으로 구성될 수 있습니다. 현재 구현에서는 URL 단위와 플레이어 진입 단위가 레슨에 가깝습니다.

## Page 또는 Step

사용자가 한 번에 보는 최소 학습 화면입니다.

기본 구조는 다음과 같습니다.

```text
좌측: 서술 영역
우측: 워크스페이스
```

페이지는 퀴즈형, 블럭 배치형처럼 고정된 타입을 갖지 않습니다. 페이지는 공통 레이아웃을 갖고, 우측 워크스페이스에 어떤 블록을 조합하느냐에 따라 경험이 달라집니다.

## Reading Pane

페이지 좌측의 서술 영역입니다.

역할:

- 맥락 설명
- 개념 설명
- 실습 지시
- 판단 기준 제시
- 읽으며 확인할 질문 제시

## Workspace

페이지 우측의 조작 영역입니다.

워크스페이스에는 이미지, 다이어그램, 퀴즈, 블럭 배치, 입력 폼, 체크리스트, 결과 카드 같은 컴포넌트가 배치됩니다.

## Workspace Block

워크스페이스 안에 들어가는 재사용 가능한 컴포넌트입니다.

예시:

- 이미지 뷰어
- 다이어그램
- 퀴즈
- 카드 분류
- 블럭 보드
- 체크리스트
- 텍스트 입력
- 프롬프트 빌더
- Before/After 비교
- 산출물 카드

Shiftbase의 저작 스펙은 페이지 타입 기반이 아니라 워크스페이스 블록 조합 기반이어야 합니다.

## Resource

학습자가 참고하거나 가져갈 수 있는 자료입니다.

예시:

- 프롬프트 템플릿
- 체크리스트
- 예제 파일
- 기준표
- 다운로드 파일

## Output

컨테이너 또는 레슨을 완료했을 때 학습자에게 남는 결과물입니다.

예시:

- 내 업무 타임라인
- AI 적용 후보 목록
- 상사에게 확인할 질문
- 업무 착수 체크리스트
- 직무별 프롬프트 세트

## State

학습자가 페이지를 진행하며 만든 입력값, 선택값, 결과물을 저장하는 데이터입니다.

예시:

- 3페이지에서 배치한 업무 블럭
- 5페이지에서 선택한 AI 적용 후보
- 8페이지에서 작성한 내 업무용 프롬프트

상태는 페이지 간 연결 학습을 가능하게 하는 핵심 개념입니다.

## Spec

컨테이너의 구성, 페이지 서술, 워크스페이스 블록, 상태 연결, 산출물을 정의하는 구조화된 저작 데이터입니다.

현재는 일부 콘텐츠가 코드에 하드코딩되어 있지만, 장기적으로는 스펙을 읽어 플레이어가 렌더링하는 구조를 목표로 합니다.
